Asp.Net Core中间件和路由

发表于 2019-05-14 | 更新于 2019-04-20 | 分类于 Web | 评论数：

注入日志中间件

public void Configure(IApplicationBuilder app,IHostingEnvironment env,IWelcomeService welcomeService,ILogger<Startup> logger)
{
  //Use在启动配置的时候走了一遍，只运行一次
  app.Use(next=>
  {
   logger.LogInformation("app.Use()......");//使用日志
   //请求没有拦截的话，请求每次都会运行
   return async httpContext=>
   {
       logger.LogInformation("----async httpContext");
       if(httpContext.Request.Path.StartsWithSegments("/first"))
       {
         logger.LogInformation("----First!!!");
         await httpContext.Response.WriteAsync("First!!!");
       }
       else
       {
         logger.LogInformation("----next(httpContext)");
         await next(httpContext);//没有访问first目录，继续往下走
       }
    };
  });

  app.Run(async(context)=>
  {
   var welcome=welcomeService.GetMessage();
   await context.Response.WriteAsync(welcome);
  });
}

使用路由

路由分为：1.Convention Route(约定路由) 2.Attribute Route(属性路由),不能混用

约定路由：

public void Configure(....)
{
    //创建wwwroot文件夹，新建index.html，使用app.UseFileServer()才能访问

    //使用默认路由配置的mvc
    app.UseStaticFiles();
    //app.UseMvcWithDefaultRoute();

    //手动配置约定路由 /Home/Index/3
    app.useMvc(builder=>
        {
            //自定义路由 （名称，模板规则）如果目录没有Home，Index，默认使用Home，Index
            builder.MapRoute("Default","{controller=Home}/{action=Index}/{id?}");
        });
}

访问网站根目录即可到达Index页面

属性路由

新建一个AboutController类

//按照路径，先访问控制器名，再访问方法
[Route("[controller]/[action]")]
public class AboutController
{
    public string Me()
    {
        return "Dave";
    }
    public string Company()
    {
        return "No Company";
    }
}

/about/me 返回Dave
